campaign gameplay (mid-mission, with active enemies and environmental

## The Run 1 outlier explained: PSO compile spike

Run 1 has `max_ms = 58.56` — a single 58 ms frame spike — while runs 3
and 5 (also baseline) have max 4.67 ms and 4.50 ms (clean).

This is almost certainly a **DirectX shader/PSO compile event** on cold
cache. MCC compiles some shaders on-demand the first time they're hit.
The first baseline run after game launch catches this; subsequent runs
use cached shaders.

Impact on metrics:

- The 58 ms spike inflates Run 1's `stutter_abs_60hz` to 1 (one frame > 16.67 ms)
- It does NOT inflate Avg, P99, or P99.9 because a single frame in
  ~10,500 total frames has negligible weight on those statistics
- Run 1 Avg = 2.85 ms is similar to Run 3 (2.75) and Run 5 (2.70) — the
  spike is one frame out of thousands

The verdict is robust: the SIGNIFICANT result holds even if Run 1 is
included; excluding it doesn't change conclusions.

## Why Halo 2 responds so strongly — engine archaeology

Halo 2 is from **2004**. That makes it 11 years older than Fallout 4 and
19 years older than Hogwarts Legacy. The remaster (Anniversary, 2014;
MCC PC, 2020) adds remastered graphics on TOP of the original engine
architecture but does NOT rewrite the simulation core.

| Property                               | Halo 2 (2004)                             | Fallout 4 (2015)                  | Hogwarts (UE4.27, 2023)        |
| -------------------------------------- | ----------------------------------------- | --------------------------------- | ------------------------------ |
| Engine origin year                     | 2004                                      | 2008 (Skyrim base) → 2015 polish | 2014 (UE4) → 2023 polish      |
| Console target gen                     | OG Xbox                                   | PS4 / Xbox One                    | PS5 / Xbox Series              |
| Single-thread reliance                 | **Extreme**                         | High                              | Moderate                       |
| Per-frame CPU work                     | Small (2004-era simulation cost)          | Large (Papyrus + streaming)       | Mixed (task graph distributes) |
| Cross-CCD penalty IMPACT               | **Severe relative to frame budget** | High                              | Moderate                       |
| Engine ceiling on 7950X3D (no Ayama)   | ~370 FPS                                  | ~178 FPS                          | ~86 FPS                        |
| Engine ceiling on 7950X3D (with Ayama) | ~571 FPS                                  | ~241 FPS                          | ~99 FPS                        |
| Observed Ayama Δ                      | **+56% FPS**                        | +35% FPS                          | +15.5% FPS                     |

The pattern is **monotonic by engine era**: older engine = bigger
Ayama benefit. This is logical — older engines have less parallelism,
more single-thread reliance, less work distributed across cores. On a
modern dual-CCD CPU, the migration penalty becomes proportionally
larger relative to the simpler per-frame work.

## Why Halo 2 outperforms Fallout 4 in delta — the physics constraint

Fallout 4 result is **conservative because of Creation Engine's
physics-FPS coupling**. The Havok-based physics solver goes unstable
above ~60 FPS without mods. Even with the "High FPS Physics Fix" mod,
VSync (or a manual cap) must remain enabled to keep the game playable
(NPCs phase through walls, doors slam, weapons spin out without it).

This means the Fallout 4 test was **CPU-thread-not-fully-saturated**:
the hot thread had spare headroom from the VSync constraint. Cross-CCD
migration penalty was partially absorbed by that headroom. Ayama still
recovered +35% FPS, but the *underlying potential* was greater.

Halo 2 has **no such constraint**. The engine runs cleanly at any
framerate. Uncapped, it sustains 367 FPS baseline (CPU-bound) and 571
FPS treated (CPU-thread-pinned-to-V-Cache, no migration). The +56%
delta reflects the engine's full CPU-bottleneck depth without
methodology constraints.

**Inference**: If Fallout 4 could be tested without the physics
constraint, expected delta would be in the +50-60% range — similar to
Halo 2. The physics constraint, not the engine architecture, is what
keeps Fallout 4's measured result at +35%.

This is a useful methodological observation for future testing of
Bethesda games (Skyrim SE, Starfield, Fallout 76) — they will all face
the same constraint and their reported deltas will be conservative.

## Caveats and limitations

1. **n=3 vs n=2 is small** — but every metric is so far outside the CI
   overlap zone that statistical formality is irrelevant. The frame
   count delta is observable directly.
2. **Single in-game scenario.** A different mission/area with heavier
   GPU load might shift the CPU/GPU balance. The scenario was chosen
   to be representative of typical Halo 2 gameplay.
3. **Run 1 baseline has a 58 ms PSO compile spike.** Documented above;
   does not affect verdict. Excluding it tightens but does not change
   the result.
4. **MCC Halo 2 remastered specifically.** Halo 2 classic graphics mode
   (the unremastered original look) might respond differently — not
   tested. The remaster adds GPU work, which actually HELPS reveal CPU
   bottleneck (GPU bound less, CPU bound more).
5. **Halo MCC is owned by Microsoft + 343 Industries.** Test conditions
   (no anti-cheat in single-player campaign) match Ayama's "safe to
   test" criteria. Multiplayer would invoke EAC or similar — out of
   scope per the user's decision to avoid AC-protected scenarios.
